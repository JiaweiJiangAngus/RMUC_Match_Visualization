# RMUC 对局预测实验

这里的第一版模型解决一个边界清楚的问题：给定最近 5 秒的全场状态，预测一辆仍存活机器人的未来位置。

模型同时输出 `1 / 3 / 5 / 10 / 15` 秒后的位置。输入包含：

- 双方 12 辆移动机器人最近 0、1、3、5 秒的位置与血量
- 双方基地、前哨站血量和剩余金币
- 目标机器人的速度、朝向、弹药变化、易伤状态和类型
- 当前比赛进度

比赛进度固定按规则时长 420 秒计算，不使用赛后才知道的实际结束时间。

蓝方样本会旋转 180° 到红方视角，因此模型学习的是阵营无关的战术规律。训练、验证、测试按整场比赛分组；同一场 BO3/BO5 的不同局不会跨集合，避免测试分数虚高。

## 训练

在项目根目录运行：

```bash
python3 ml/train_trajectory.py
```

默认使用 CUDA（存在时），最多抽取 30 万训练样本。模型和完整评估报告分别保存到：

```text
ml/artifacts/trajectory_mlp.pt
ml/artifacts/trajectory_mlp.metrics.json
```

快速验证数据管道可以缩小样本和轮数：

```bash
python3 ml/train_trajectory.py \
  --epochs 2 \
  --max-train-samples 20000 \
  --max-val-samples 5000 \
  --max-test-samples 5000
```

报告会把模型与两个基线比较：

- `stationary`：假设机器人停在原地
- `constant_velocity`：按最近 3 秒速度直线外推
- `learned_mlp`：本模型

位置误差之外，`zone_accuracy` 把 28 m × 15 m 场地划成 4 m × 3 m 的战术格，衡量“下一步会去哪一片区域”。

## 当前基线结果

当前检查点使用 30 万训练样本，在按整场比赛隔离的 90 局、68,716 条测试样本上评估。下表是平均位置误差（米）；“移动中”表示目标最近 3 秒速度不低于 0.15 m/s。

| 预测跨度 | 原地基线（全量） | 模型（全量） | 原地基线（移动中） | 模型（移动中） |
| ---: | ---: | ---: | ---: | ---: |
| 1 秒 | 0.394 | 0.305 | 0.756 | 0.521 |
| 3 秒 | 0.912 | 0.731 | 1.776 | 1.301 |
| 5 秒 | 1.331 | 1.100 | 2.487 | 1.886 |
| 10 秒 | 2.092 | 1.789 | 3.611 | 2.834 |
| 15 秒 | 2.607 | 2.272 | 4.265 | 3.412 |

5 秒移动目标的 4 m × 3 m 战术格命中率为 52.69%，原地基线为 45.83%。完整的均值、中位数、P90、1/2 米内比例和区域命中率见 `artifacts/trajectory_mlp.metrics.json`。

## 调用模型

指定一局和一个时刻，可以预测当时场上所有移动机器人：

```bash
python3 ml/predict_trajectory.py --game-id 1780037479635 --second 200
```

也可以只看一辆车：

```bash
python3 ml/predict_trajectory.py \
  --game-id 1780037479635 \
  --second 200 \
  --side 红 \
  --robot-type 哨兵
```

如果预测时刻之后的真实帧仍在该局数据中，输出会一并给出真实坐标和误差，便于回放对照。

这还是行为预测模型，不是可做反事实推演的完整环境。后续要训练决策策略，应先在它之上增加事件/伤害预测、资源状态转移和不确定性分支，再做离线模仿学习或保守离线强化学习。

## 下棋式对局模拟

网页底部的“对局模拟”把一局 420 秒切成 28 个 15 秒回合。每队每个回合会从真实比赛样本中抽取并融合：

- 基地、前哨站和地面机器人的伤害；
- 17mm、42mm 与飞镖伤害构成；
- 飞镖命中/开闸窗口；
- 己方与敌方堡垒附近的驻留时间；
- 能量机关增益和官方记录的地形动作。

网页默认展示 TDT 对 RPS，可逐回合落子、自动演算单局，或批量模拟 500 局。跨赛区对阵属于反事实推演，批量结果是用于比较战术风格的实验分布，不是官方胜率。

底层静态棋谱由区域赛 SQLite 数据生成：

```bash
python3 ml/build_match_simulation_data.py
```

输出文件为 `docs/data/models/match_simulation.json`。堡垒控制目前通过存活地面机器人在堡垒中心附近的驻留秒数推断，因为事件表中没有独立的裁判占领事件。

## 操作手战术推理

`tactical_inference.py` 把坐标预测转成客户端可直接显示的内容：

- 5/10/15 秒候选战术区域，10 秒为主目标；
- 预计经过的 B/R 地形入口和路线摘要；
- 对照人工确认/官方事件/轨迹证据的能力风险提示；
- 全场状态模型与 44 队的学校×兵种×区域×比赛阶段先验融合。

构建/更新战队先验：

```bash
python3 ml/build_tactical_priors.py
```

回放某一时刻的操作手输出：

```bash
python3 ml/tactical_inference.py \
  --game-id 1778680444314 --second 200 \
  --side 红 --robot-type 步兵3
```

本地 HTTP 服务：

```bash
python3 ml/tactical_api.py --host 127.0.0.1 --port 8765
```

`POST /infer` 只需提供当前 `second`、红蓝学校信息和最近 5 秒帧。帧内每个机器人使用导出数据的紧凑数组格式：

```text
[id,type,side,hp,max_hp,x,y,yaw,ammo17,ammo42,coins,vulnerable]
```

推理器使用 MC dropout 生成候选分布；`display_confidence` 还会乘以独立测试集在对应时距上的历史战术格命中率。它适合做提示强度，不应解释为严格的贝叶斯概率。

### 在比赛回放中演示

直接启动当前 PyQt 回放客户端：

```bash
python3 app.py
```

选择赛区、场次和局次后，点击时间轴右侧的“开启预测”。地图会显示：

- 红/蓝虚线：从当前位置到未来 10 秒的粗轨迹；
- 圆环：10 秒主目标点；
- 标签：兵种、目标战术区域和显示置信度；
- 右下状态：当前输出的存活地面机器人数和单次推理耗时。

拖动时间轴或正常播放时会自动请求新的整秒预测。模型在后台线程运行，不阻塞时间轴；阵亡机器人不绘制未来路线。
