# RMUC 2026 区域赛数据可视化

本地 PyQt5 可视化窗口，直接读取桌面上的 RMUC 2026 区域赛 SQLite 数据集。

## 启动

双击桌面上的“RMUC 2026 数据可视化”，或在终端运行：

```bash
./run.sh
```

也可以指定另一份同结构数据库：

```bash
./run.sh --db /path/to/dataset.sqlite
```

## 手机 / Windows 模式

在 Qt 主窗口顶部点击“手机 / Windows”，程序会启动响应式 Web 界面并打开本机浏览器。
Web 首页入口是项目根目录的 `index.html`；页面通过 `web_app.py` 读取 SQLite 数据，因此请通过下面的服务地址访问，不要直接使用 `file://` 打开。

- 本机访问：`http://127.0.0.1:8876`
- 手机或其他 Windows 电脑：使用弹窗显示的局域网地址，例如 `http://192.168.43.34:8876`
- 手机、Windows 电脑与运行服务的电脑需要连接同一局域网
- 手机端会自动改为单列触控布局；Windows 端会自动使用宽屏三栏布局
- Windows 上若已复制完整数据目录，也可以双击 `start_web.bat` 独立启动

Linux 上也可在终端直接启动 Web 版：

```bash
./start_web.sh
```

若其他设备无法访问，请允许系统防火墙放行 TCP 端口 `8876`。

## 窗口内容

- 按东部、南部、北部赛区筛选场次与局次
- 红蓝双方、胜方、比赛时长与事件数量概览
- 逐秒战场俯视图、最近 20 秒机器人运动轨迹
- 直接复用 `TDT/Client/resource/Map.png` 的 2026 战场地图
- 基地与前哨站位置沿用 TDT Client QtUI 的固定坐标和状态标记
- 主画面采用 TDT Client 风格：左侧红方 HUD、中央大地图、右侧蓝方 HUD
- 回放时对相邻秒坐标进行插值，机器人在地图上平滑移动
- 机器人血量、基地/前哨站状态和队伍剩余金币
- 顶部使用红蓝对称血条显示双方基地、前哨状态
- 飞镖闸门/命中/伤害与雷达标记/UAV 反制按双方独立统计
- 地图标记随窗口缩放；英雄/工程/步兵/哨兵/无人机显示为 `1/2/3/4/AI/6`
- 可点击的全局事件时间线与最近事件列表
- 播放、暂停、前后跳转及 0.5×～4× 回放速度

程序按局查询，不会一次性把约 1.2 GB 的 SQLite 数据全部载入内存。
