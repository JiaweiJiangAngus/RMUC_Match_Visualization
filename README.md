# RMUC 2026 区域赛数据可视化

本地 PyQt5 可视化窗口，直接读取桌面上的 RMUC 2026 区域赛 SQLite 数据集。

## 启动

双击桌面上的“RMUC 2026 数据可视化”，或在终端运行：

```bash
./run.sh
```

也可以指定另一份同结构数据库：

```bash
./run.sh --db /path/to/dataset.sqlite
```

## 本地 Web 开发模式

项目根目录的 `index.html` 是连接 SQLite API 的本地开发版。启动方式：

```bash
./start_web.sh
```

然后访问 `http://127.0.0.1:8876`。Windows 本地开发可运行 `start_web.bat`。

## GitHub Pages 静态版

`docs/` 中是可直接发布的完整静态站点，包含全部 613 局比赛。比赛数据按局保存为 gzip 文件，页面只在切换对局时下载对应文件，不需要运行 Python 后端。手机直接打开 GitHub Pages 地址即可使用，并提供底部触控导航、地图全屏和手机优先的信息顺序。

页面默认开启浏览记忆，会立即保存当前赛区、场次和局次 `game_id`，并自动恢复回放位置、播放速度、主题和背景。顶部“记忆 开/关”按钮可以随时关闭并清除比赛浏览记录；每个浏览器最多保留最近 50 局的回放位置。

数据集更新后重新生成：

```bash
python3 export_pages.py
```

静态站前端的权威源文件位于 `docs/`，修改 HTML、CSS 或 JavaScript 后可直接本地检查并提交。如果只需刷新共享地图和 Pages 元数据，可运行：

```bash
python3 export_pages.py --frontend-only
```

本地检查静态站点：

```bash
python3 -m http.server 8000 --directory docs
```

然后打开 `http://127.0.0.1:8000`。发布时提交 `docs/`，再到 GitHub 仓库的 `Settings → Pages` 选择：

```text
Source: Deploy from a branch
Branch: main
Folder: /docs
```

当前 `docs/` 总大小约 45 MB，其中分局压缩数据约 39 MB；不要把原始 1.2 GB SQLite 文件提交到仓库。

## 窗口内容

- 按东部、南部、北部赛区筛选场次与局次
- 红蓝双方、胜方、比赛时长与事件数量概览
- 逐秒战场俯视图、固定保留最近 5 秒机器人运动轨迹
- 直接复用 `TDT/Client/resource/Map.png` 的 2026 战场地图
- 网页使用同尺寸 WebP 地图资源，在保持地图清晰度的同时减少首屏下载量
- 基地与前哨站位置沿用 TDT Client QtUI 的固定坐标和状态标记
- 主画面采用 TDT Client 风格：左侧红方 HUD、中央大地图、右侧蓝方 HUD
- 回放时对相邻秒坐标进行插值，机器人在地图上平滑移动
- 纵向按整张地图约 17m 校准，将机器人 `0–15m` 坐标映射到地图中央 `15/17` 区域
- 机器人血量、基地/前哨站状态和队伍剩余金币
- 顶部使用红蓝对称血条显示双方基地、前哨状态
- 飞镖闸门/命中/伤害与雷达标记/UAV 反制按双方独立统计
- 地图标记随窗口缩放；英雄/工程/步兵/哨兵/无人机显示为 `1/2/3/4/AI/6`
- 机器人图标下方显示 17mm/42mm 累计已发数（工程机器人不显示）
- 无人机被雷达反制后显示 45 秒黄色外圈，多次反制叠加持续时间
- 可点击的全局事件时间线与最近事件列表
- 播放、暂停、前后跳转及 0.5×～4× 回放速度

程序按局查询，不会一次性把约 1.2 GB 的 SQLite 数据全部载入内存。
